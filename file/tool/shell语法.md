## 执行脚本的方式

- ( 初级方式 )
- 首先要将编写的.sh文件变成可执行文件，命令：`chown +x [文件名]`

- 然后直接在文件所在目录下执行：`./[文件名]`



- 熟练方式
- 直接将.sh 文件放在环境变量中该用户bin的路径下，这样在切换到该用户对服务器进行操作的时候，可以在任何位置直接打文件名进行shell 文本执行。





## Shell 变量

>  在 Shell 编程中，变量是用于存储数据值的名称。定义变量时，变量名不加美元符号（$，PHP语言中变量需要）,例如：`your_name="runoob"`

### 遵守的命名规则（这里不举例说明）
- 只包含字母、数字和下划线
- 不能以数字开头
- 避免使用 Shell 关键字
- 使用大写字母表示常量，例如 `PI=3.14`
- 避免使用特殊符号
- 避免使用空格


### 特别注意

- 等号两侧避免使用空格
```shell
# 正确的赋值
variable_name=value

# 有可能会导致错误
variable_name = value
```

### “$” 符号的应用
- 变量的取值：`HOME="Hello World!"  echo ${HOME}` => 输出“Hello World!”
- 位置参数: `$0 - $9` 脚本或函数的参数。$0是脚本名，$1-$9是第1到第9个参数。举例：执行 `./script.sh arg1`，则$0是`./script.sh`，$1是`arg1`
- `$#` : 参数个数 (传递给脚本或函数的参数总数) 。举例：若`./script.sh arg1 arg2`，则$#为2
- `$?` : 退出状态：上一条命令的退出状态，0通常表示成功。举例：ls /tmp; echo $?（成功则输出0）


### 关系运算符

- `-eq` : equal to 等于
- `-ne` : not equal to 不等于
- `-lt` : less than 小于
- `-le` : less than or equal to 小于等于
- `-gt` : greater than 大于
- `-ge` : greater than or equal to 大于等于


### 运算符

|运算符 |含义与用途说明 |示例（可直接在脚本中使用）|
|:----:|:----:|:----:|
-d | 检查指定路径是否存在且为一个目录。| [ -d "/tmp" ] && echo "是目录"|
-f |检查指定路径是否存在且为一个常规文件（非目录、设备文件等）| [ -f "report.txt" ] && echo "是文件"|
-e | 检查指定路径的文件或目录是否存在（不区分类型）| [ -e "/some/path" ] && echo "路径存在"
