### Flink ON Yarn 的两种方式

- 在yarn上预先初始化一个Flink集群，占用Yarn中固定的资源。不管Flink集群有没有执行任务，都会占用Yarn的资源，除非手动停止
> bin/yarn-session.sh -n 2 -jm 512 -tm 512 -d


- 每次提交Flink的任务时才会向Yarn申请资源，即每次提交任务都会在Yarn上创建一个新的Flink集群，任务停止后，Yarn自动将资源收回
> bin/flink run -m yarn-cluster -yn 2 -yjm 800 -ytm 800 /export/servers/flink-1.6.0/examples/batch/WordCount.jar（旧版本中 -yn这个已经被弃用）

`bin/flink run \
  -t yarn-per-job \
  -Djobmanager.memory.process.size=800m \
  -Dtaskmanager.memory.process.size=800m \
  -Dtaskmanager.numberOfTaskSlots=2 \
  -Dyarn.containers=2 \
  /opt/module/flink/examples/batch/WordCount.jar`

上边的方法报内存的错误：这个错误是因为内存配置冲突导致的。Flink 计算出的各个内存部分总和超过了配置的总内存。让我为你提供一个完整的内存配置方案：# 只设置进程总内存，让Flink自动分配

  bin/flink run -t yarn-per-job \
  -Djobmanager.memory.process.size=1024m \
  -Dtaskmanager.memory.process.size=2048m \
  -Dtaskmanager.numberOfTaskSlots=2 \
  /opt/module/flink/examples/batch/WordCount.jar
